## Introducción

Cuando escuché por primera vez sobre <ExternalLink href='https://git-scm.com/'text='Git' />, pensé que era algo sencillo: una herramienta para **“guardar versiones del código”**. Pero cuando empecé a usarlo de verdad, me di cuenta de que no era tan fácil como parecía. Entre ramas, commits, merge, stash, y conflictos... me confundí más veces de las que quisiera admitir.

Hoy ya no me da miedo usarlo. Lo uso todos los días y me salvó más de una vez. Así que escribí este artículo para compartir lo que aprendí: qué es Git, por qué es tan útil, cuáles son los comandos más importantes y cómo fui perdiéndole el miedo con la práctica.

## ¿Qué es Git?

<ExternalLink href='https://git-scm.com/'text='Git' /> es un sistema de control de versiones distribuido. Eso significa que te permite llevar el control de todos los cambios que hacés en tu proyecto, de forma local (en tu máquina) y también remota (por ejemplo en <ExternalLink href='https://github.com/git'text='GitHub' /> o <ExternalLink href='https://about.gitlab.com/'text='GitLab' />).

Te sirve para:

Es una herramienta creada por <ExternalLink href='https://es.wikipedia.org/wiki/Linus_Torvalds'text='Linus Torvalds' /> (sí, el mismo que creó Linux) y se volvió un estándar en el desarrollo moderno.

## ¿Por qué me costó tanto?

Porque al principio usaba Git **como si fuera una caja negra**. Veía los comandos, los copiaba, y listo. Pero si algo salía mal, no sabía qué había hecho ni cómo arreglarlo.

También me perdía con las ramas. ¿Dónde estoy parado? ¿Estoy haciendo commit en la rama correcta? ¿Qué pasa si hago merge? ¿Y pull? Todo eso me parecía confuso.

Lo que me ayudó fue entender qué hace cada comando, y dejar de depender solo de la interfaz del IDE. Hoy sigo usando VS Code o IntelliJ para cosas básicas, pero ya no tengo miedo de abrir la terminal y resolver cosas desde ahí.


## Mis comandos favoritos de Git (y para qué sirven)

Acá te dejo los comandos que más uso, explicados de forma sencilla. Todos los podés practicar en un repo personal.

### 1. **git init**  
Inicializa un nuevo repositorio Git en tu carpeta actual.

<CodeBlock code={`
git init`} />

### 2. **git status**  
Te muestra en qué rama estás y qué archivos cambiaron.

<CodeBlock code={`
git status`} />

### 3. **git add .**  
Agrega todos los archivos modificados al área de preparación (staging).

<CodeBlock code={`
git add .`} />

### 4. **git commit -m "mensaje"**  
Guarda los cambios con un mensaje explicando qué hiciste.

<CodeBlock code={`
git commit -m "Agregué nueva funcionalidad"`} />

### 5. **git log --oneline --graph**  
Muestra un historial visual de los commits.

<CodeBlock code={`
git log --oneline --graph`} />

### 6. **git checkout -b nueva-rama**  
Crea y te cambia a una nueva rama.

<CodeBlock code={`
git checkout -b nombre-de-la-rama`} />

### 7. **git merge nombre-rama**  
Une los cambios de otra rama con la actual.

<CodeBlock code={`
git merge nombre-de-la-rama`} />

### 8. **git pull**  
Descarga y aplica los cambios del repositorio remoto.

<CodeBlock code={`
git pull`} />

### 9. **git push**  
Sube tus commits al repositorio remoto.

<CodeBlock code={`
git push`} />

### 10. **git stash**  
Guarda temporalmente tus cambios sin hacer commit.

<CodeBlock code={`
git stash`} />

##  Git es más que solo comandos
Git es una herramienta poderosa. No solo te permite guardar tu código, sino también trabajar de forma ordenada, colaborativa y segura.
Lo que vimos acá es solo la base: suficiente para comenzar, pero no para quedarse ahí.

Si querés seguir aprendiendo y entender mejor cómo trabajar con Git en equipos, te recomiendo leer sobre <ExternalLink href='https://www.atlassian.com/es/git/tutorials/comparing-workflows/gitflow-workflow' text='Git Flow' />, un modelo de ramificación muy utilizado en proyectos reales. Te va a ayudar a organizar el desarrollo en ramas como `main`, `develop`, `feature`, `release`, y más.
